단일고장 베이스(Passive-FTC) + IRIS 실험 2단계 완료

베이스/드론 검증

  • Passive-FTC 단일고장 검증: 모터 2번이 t=1초에 영구 완전 고장(꺼졌다 켜지는 게 아니라 그 뒤로 계속 정지 = 4모터 중 1개 완전 상실) → 25초 제어비행, 최종고도 −0.06m(추락 없음). 옛 코드(이중 전용)는 단일고장 시 axis_tilt·chi 전 범위 추락(고도 기준 재확인) → 전환 정당성 확정.
  • 스핀 안정화 확인: 고장 후 요(yaw) 각속도가 −800°/s로 수렴(일정 스핀), 누적 요각 직선 증가(약 55회전). Mueller 2014 relaxed-hover와 동일 메커니즘.
  • 드론 IRIS(1.37kg) 교체: Minnie(0.29kg) 너무 작아 → 더 큰 IRIS로. init 한 줄로 교체, 제어기 자동 재유도. 단일고장 생존 확인(강하 −6.2m→회복).
  • 무화물 baseline 확정: Agility 0.5/0.5. 강하 5.4m·회복 7.3s. (IRIS는 무거워 강하 큼 = 본질적, 적응 효과 보일 여지 큼)

★ 화물 시험 — 문제 입증 (baseline B, 제어기는 화물 모름)

화물을 plant에만 주입: 질량 + 평행축 관성 + 무게중심 하강(CoG_Pos_c). 제어기는 마른 IRIS 기준 고정.

핵심: 임계점 200~500g 사이. 기존 FTC(화물 모름)는 500g부터 추락. IRIS 3로터로 1.87kg 들 추력은 있으므로 추력부족이 아니라 제어 실패(질량 오인 → 추력배분 오류) → 적응형 FTC가 살릴 여지 있음. "기존 FTC는 화물 500g에서 추락" = 연구 출발점.

★ 무게 추정기 새 모델 이식 — 성공 (test_estimate_IRIS.m)

▶ 가설(실험 전): 호버에서 총추력은 중력과 평형 → Σ추력 ∝ 총질량. 새 모델 추력 ≈ k·Σ(모터속도²). 따라서 마른 드론으로 1회 보정하면 화물 호버 모터속도만으로 총질량을 역산 가능할 것이다: m_total = m_dry · sum(ω_hover²)/sum(ω_dry²) (PX4 호버추력 추정과 동일 원리).

▶ 실험 방법: 마른 IRIS 호버로 Σω²_dry 보정 → 화물 0/200/500/1000g을 plant에만 장착(제어기는 모름) → 고장(t=1s) 전 안정 호버 구간(−0.5~0.8s)에서만 모터속도 측정 → m_total−m_dry를 참값과 비교.

▶ 검증 결과:

시뮬레이션 구조 파악

  • 데이터 로깅: 모델이 accB(IMU 비력), wRotorMeas(로터속도), forces 등을 워크스페이스로 자동 저장(Timeseries).

2단계 — 화물 무게 추정 검증 완료

셋업

  • 화물 주입(simParams.m): plant에만 화물 반영, 제어기/추정기는 모름. payload.mass(0/0.2/0.5/1.0kg), payload.offset=0.05m. 총질량 = 드론(0.375)+화물, 관성은 평행축 정리로 보정.
  • 호버 구간 확보(controlParams.m): fail_time 0 → 5.0초. 0–5초 정상 호버(측정 구간) → 5초 로터 1,3번 고장 → 스핀 안정화.

핵심 물리 검증 (연구 동기 입증)

  • 화물 0g: 고장 후에도 고도 유지 (정상 비행)
  • 화물 200g: 고장 후 고도 하락 → "화물 모르는 기존 FTC는 고장 시 화물 때문에 추락" 재현 성공
→ 적응형 FTC의 필요성을 시뮬레이션으로 입증. (최종 보고서 핵심 그래프 후보)

무게 추정 알고리즘 (estimate_payload.m)

원리: 수평 호버에서 IMU 비력 accB_z = -T/m → m = T / abs(accB_z). 추력 T는 측정 로터속도로 독립 계산 T = Σ k0·wᵢ². 화물무게 = 추정 총질량 − 드론 자체무게(0.375). 방식: 후처리(post-processing), PX4 Hover Thrust Estimator 로직 참조.

통찰

추정 구간을 2.0–4.5초로 넓게 잡으면 오차 16.7%(233g), 안정 호버 초반 2.0–2.5초로 좁히면 오차 3.3%(207g). 추정 정확도는 호버 안정성에 민감 → 안정 구간 선택이 핵심. (계획서의 "안정된 호버 구간에서 1회 추정" 방침의 근거)

실험 설계 (가설 · 검증 방법 · 검증 결과)

가설

  • 주가설(H): 화물 무게를 추정하여 제어기의 파라미터(질량·관성)와 게인·허용기울기를 화물 조건에 맞게 적응시키면, 고정 파라미터 기존 FTC가 추락하는 화물 조건에서도 안정 비행(생존)이 가능하다.
  • H1: 질량(+관성)만 올바르게 반영해도 추락을 막을 수 있는가? (INDI 추력배분 G행렬 보정만으로 충분한가)
  • H2: 질량만으론 부족하고, 스핀 중 무게중심-원심력에 의한 기울기 심화 때문에 게인·허용기울기(par.chi, par.position_maxAngle)까지 적응해야 하는가?

→ H1/H2 중 무엇이 맞는지 가리는 것 자체가 핵심 연구 결과. (현재 미확정)

검증 방법

  • 비교 대상 (동일 화물·동일 시나리오, 제어기만 교체):
    • B — 기존 FTC: 질량 0.375 고정, 게인 고정 (화물 모름)
    • C1 — 적응형(부분): 추정 질량·관성만 반영
    • C2 — 적응형(전체): 질량·관성 + 게인 + 허용기울기 모두 적응
    • (계획서 4단계의 고정 PID는 하한 기준선으로 추가)
  • 공통 시나리오: 0–5초 호버(이 구간 무게 추정) → 5초 로터 1,3 고장 → 스핀 안정화
  • 화물 조건: 0 / 200 / 500 g (1000g은 추력 포화 가능성 별도 확인)
  • 평가 지표: ① 생존 여부(추락/안정) ② 최대 기울기(롤·피치) ③ 고도 손실/오버슛 ④ 정착 시간 ⑤ 최대 스핀 각속도
  • 전제(완료): 무게 추정 정확도 — 200g→3.3%, 500g→1.8%. 적응에 쓸 추정값 신뢰 가능.

simParams.m — 41행 뒤 화물 주입 블록 추가

%% Payload (적응형 FTC 연구용)
payload.mass   = 0.5;     % [kg] 진짜 화물무게 (실험: 0 / 0.2 / 0.5 / 1.0)
payload.offset = 0.05;    % [m]  하단 고정 오프셋
simVars.payload = payload;
simVars.drone.massDry = loadedParams.parameters.m;          % 드론 자체무게(추정 기준)
simVars.drone.mass = simVars.drone.massDry + payload.mass;  % 총질량 = 드론 + 화물
simVars.drone.Iv   = simVars.drone.Iv + payload.mass*payload.offset^2*diag([1 1 0]); % 평행축

→ plant(EOMQuat)가 읽는 simVars.drone.mass/.Iv에 화물 반영. 제어기는 모름.
