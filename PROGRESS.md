**과제명:** 화물 장착 드론의 로터 고장 대응 적응형 제어기 설계 (Adaptive FTC for Quadrotors with Bottom-Fixed Payload under Rotor Failure)
**기반 코드:** Passive-FTC-Quadrotor (Beyer et al., AIAA JGCD 2023, iff-gsc) — INDI + 제어배분 기반 단일 로터 완전 고장 대응
**드론 모델:** IRIS (1.37 kg, 쿼드로터) · **시뮬레이션:** MATLAB/Simulink R2026a
**연구원:** 박주호 | **지도교수:** 임헌국

| 단계 | 내용 | 상태 |
|------|------|------|
| 1단계 | 오픈소스 실행 환경 구축, 파라미터 구조 파악 | ✅ 완료 |

## 실험 ① 화물 질량 추정 ✅ (오차 ≤ 2%)

**▶ 가설:** 호버에서 총 추력 = 중력이므로 추력 ∝ 총질량. 추력 ≈ k·Σ(모터속도²)이므로, 무화물로 1회 보정하면 호버 모터속도만으로 총질량(→화물)을 역산할 수 있다.
→ `m_total = m_dry · Σω²_호버 / Σω²_무화물`

**▶ 방법:** 무화물 IRIS 호버로 Σω²_무화물 보정 → 화물 0/200/500/1000 g 장착(제어기는 모름) → 고장 직전 안정 호버(t = −0.5~0.8 s) 모터속도로 역산.

**▶ 검증 결과:**

| 실제 화물 | 추정 화물 | 오차 |
|------|------|------|
| 0 g | 0 g | 0 % |
| 200 g | 198 g | 1.0 % |
| 500 g | 494 g | 1.2 % |
| 1000 g | 982 g | 1.8 % |

→ 전 조건 **오차 2% 이내.** 적응 제어기에 입력할 신뢰성 있는 추정값 확보. (`test_estimate_IRIS.m`)

## 실험 ② 추력 물리한계 — 무거운 화물은 적응으로 못 살림 ⚠️

**▶ 배경/가설:** 초기 목표는 "기존 제어기로 추락하는 500 g를 적응으로 생존시킨다"였다. 그러나 여러 적응 시도가 모두 추락 → 원인이 제어 실패인지 물리 한계인지 규명한다.

**▶ 방법:** 화물을 늘리며 고장 후 남은 로터 최대 회전수·모터 명령 포화율 측정 + 시작 고도(15/30/50/80 m)를 바꿔 강하량 비교.

**▶ 검증 결과:**

| 화물 | 고장 후 최대 회전수 | 명령 포화율 | 결과 |
|------|------|------|------|
| 0 g | 751 | 0 % | 생존 |
| 350 g | 765 | 7 % | 생존 |
| 500 g | 774 | **74 %** | **추락** |
| 1000 g | 775 | 45 % | 추락 |

- 회전수가 **약 775에서 천장 도달**, 500 g부터 명령이 시간의 74% 포화 = **추력 포화.**
- **시작 고도를 바꿔도 강하량 = 시작 고도** → 공중 회복 불가, 지면까지 낙하 = **과도 현상이 아닌 영구 추락**(로터 3개로 1.87 kg을 정적으로도 못 듦).

→ 500 g 추락은 **제어 실패가 아니라 추력의 물리적 한계.** 질량 추정·게인·고도 그 무엇으로도 살릴 수 없음. (`diag_thrust_IRIS.m`, `diag_altitude_IRIS.m`)

## 실험 ③ 현실적 화물 범위로 연구 재정립

**▶ 방법:** 위 한계를 실제 배송 드론 사양과 대조.
- 일반 배송(구글 Wing, Matternet): 화물 = 자체 무게의 **20~30 %**
- 추력대중량비(TWR) ≈ 1.5~2.5로 추력 여유 작음. 내결함성 배송 드론은 헥사·옥토로터 사용.

**▶ 결정:** 기체는 실재 사양 IRIS 유지, 화물을 **자체 무게의 25 % 이하(≈340 g)** 로 제한. 이 범위는 ① 실제 일반 배송 비율과 일치, ② IRIS 단일고장 생존 범위(~350 g) 내.
→ 연구 목표를 **"추락 방지"** 에서 **"생존 범위 내 고장 후 회복 성능 개선"** 으로 재정의.

## 실험 ④ 화물 물리 모델의 엄밀성 검증

**▶ 가설/동기:** "하부 고정 화물을 제대로 모사했는가, 단순히 질량만 늘린 것인가?"

**▶ 방법:** 하부 화물이 바꾸는 세 요소 — ① 질량 ② 회전 관성(평행축) ③ 무게중심(CoG) 수직 이동 — 을 켜고 끄며 동역학 비교.

**▶ 검증 결과:**
- **단순 질량 증가가 아님:** 화물에 의한 **롤·피치 관성 증가가 실제 동역학에 작용**. 관성 항은 **환산질량 μ = m_dry·pm/M** 기준 평행축으로 정밀 보정.
- **무게중심의 수직 이동은 자유 비행 회전 동역학에 무영향:** 추력이 기체 z축이고 CoG 이동도 같은 축 → 모멘트 상쇄, 중력은 CoG에 작용해 모멘트 0(진자식 복원은 고정 피벗이 있어야 성립, 자유 비행엔 피벗 없음).
- 따라서 **하부 고정 화물의 동역학적 효과 = 질량 증가 + 롤·피치 관성 증가**, 둘 다 정확히 반영됨.

→ **모델은 물리적으로 타당.** (계획서 배경의 "원심력 기울기 심화"는 자유 비행에선 성립하지 않음 — 교수님과 동기 논리 정리 필요.) (`check_payload_phys_IRIS.m`)

## 실험 ⑥ 적응 방법 비교 (핵심 결과) — 가 / 나 / 다 + 베이지안 최적화

**▶ 비교 구도:** 실제 배송 드론은 호버 추력 학습으로 총질량을 이미 대략 안다. 따라서 기여는 "무게를 추정하느냐"가 아니라 **"추정 무게를 어떻게 활용하느냐"**.
- **가 — 기존 FTC:** 화물 미반영, 게인 고정(0.5/0.5).
- **나 — 기존 + 무게추정:** 추정 질량·관성 반영, 게인 고정. *(실제 드론 수준 = 공정한 기준선)*
- **다 — 기존 + 무게추정 + 게인전환:** 위에 더해 화물별 최적 게인 적용. *(제안)*

**▶ 비용 함수 J 재정의(강하 포함):** 초기 J는 최대 강하량(고장 직후 순간 하강)을 포함하지 않아, 게인을 공격적으로 키워 "강하는 크지만 정착은 빠른" 경우(지면 충돌 위험)를 걸러내지 못했다. 이를 보완해 **4지표(최대강하·회복시간·말기틸트·최종오차)를 (가) 기준으로 정규화한 뒤 가중평균**하도록 재정의.
> J = mean( [강하/강하_가, 회복/회복_가, 틸트/틸트_가, 오차/오차_가] ) (가 = 1.0, 낮을수록 우수)

**▶ 게인 탐색 = 베이지안 최적화(Bayesian Optimization):** 격자 탐색(미리 정한 후보 전수 평가) 대신 `bayesopt` 사용. 평가한 점들로 비용의 대리 모델(가우시안 과정)을 만들어 "안 가 본 게인의 J 예측 + 불확실성"을 추정하고, 획득 함수(expected improvement)로 *유망하면서 안 가 본* 게인을 자동 선택. 적은 평가(30회)로 연속 최적점 도출. (시뮬레이션은 센서 노이즈가 없어 결정론적임을 확인 → 1회 평가로 충분.)

**▶ 검증 결과 (화물 340 g, 낮을수록 우수):**

| 제어기 | 최대강하[m] | 회복시간[s] | 말기틸트[°] | 최종오차[m] | 종합 J(가=1) |
|------|------:|------:|------:|------:|------:|
| 가: 기존 | 6.51 | 9.18 | 27.5 | 0.34 | 1.000 |
| 나: +무게추정 | 6.37 | 8.48 | 17.6 | 0.19 | 0.775 |
| **다: +무게추정+게인전환** | **6.25** | **7.20** | **4.8** | **0.042** | **0.511** |

- BO 최적 게인 = **[AgilityAtti 0.437, AgilityPos 0.724]**.
- **다가 네 지표 모두에서 최우수**(강하량까지 최소). 종합 J는 가 대비 **약 49 % 개선**, 공정 기준선 나(0.775)보다도 우수.
- 핵심 통찰: 자세 게인을 무리하게 키우면 순간 강하가 커진다. BO는 **자세 게인은 낮추고(0.44) 위치 게인은 높이는(0.72)** 조합으로 강하·회복·자세를 동시에 개선.
- **결론 수정 근거:** 초기에는 "질량·관성 반영만으로 충분"으로 판단했으나, ④의 물리 모델 정밀화 + 강하 포함 J + BO 재최적화 결과 **게인 적응 병행이 최적**임을 확인. (`optimize_gain_bo_IRIS.m`, `demo_compare3_IRIS.m`)

## 실험 ⑦ 센서(RPM) 노이즈 하 추정 강건성

**▶ 가설:** 추정기는 호버 모터속도 제곱합(Σω²)을 쓴다. RPM 측정 노이즈(백-EMF 영점교차 지터·정류 양자화·EMI·진동 등)가 있어도, 호버 구간 다중 샘플 평균이 노이즈를 억제해 추정이 강건할 것이다.

**▶ 방법:** 깨끗한 호버 회전수에 가우시안 노이즈(값의 1/2/5/10 %)를 주입하고 50회 실현하여 화물별 추정오차의 평균·표준편차 측정.

**▶ 검증 결과:** 노이즈 10 %(현실 ESC 텔레메트리 ~1~2%보다 보수적)에서도 추정 화물오차의 표준편차는 ±7~11 g에 그침. 평균 오차는 노이즈와 무관(물리모델 계통오차 ~2 %)하게 유지.
→ **다중 샘플 평균이 노이즈를 √N로 억제** = 추정기 노이즈 강건. 또한 추정기는 **IMU(가속도·자이로)를 쓰지 않으므로 IMU 노이즈에는 본질적으로 면역.** (`demo_estnoise_IRIS.m`)

> 참고: 베이스 오픈소스에는 센서 노이즈가 구현되어 있지 않음을 확인(계획서는 "내장 노이즈 활성화"를 가정했으나 미구현). 추정 검증은 위 후처리 노이즈로 수행. 제어 루프 IMU 노이즈(검증된 Aerospace Blockset / `imuSensor` 활용)는 후속 과제.

## 종합 결론

1. 드론은 **화물 질량을 오차 2 % 이내로 추정**할 수 있고, 추정은 RPM 노이즈에 강건하다.
2. **로터 1개 완전 고장 + 과대 화물(≈400 g↑)** 은 어떤 제어로도 살릴 수 없다 — 추력의 물리적 한계를 정량 규명.
3. **현실적 화물 범위(≤25 %)** 내에서, **질량·관성 반영 + 화물별 게인 적응(베이지안 최적화)** 을 결합하면 고장 후 회복 성능이 네 지표(강하 포함) 모두에서 향상(가 대비 J 49 % 개선)되고 무거운 화물의 **말기 자세 발산도 안정화**된다.
4. 화물 물리 모델의 **타당성을 검증**했다(질량 + 롤·피치 관성, z-CoG 무영향).

**연구의 가치:** "적응했더니 됐다"에 그치지 않고, **물리적 한계를 정직하게 규명**하고 현실 조건으로 재정립한 뒤, **무엇을 비용으로 최적화하느냐(강하 포함)** 까지 다듬어 실제로 효과 있는 적응 방법을 원리적 최적화(BO)로 가려냈다.
