## 2단계 — 화물 무게 추정 🔄 (2026-06-15)

### 셋업
- **화물 주입 (`simParams.m`):** plant에만 화물 반영, 제어기/추정기는 모름.
  - `payload.mass` (실험변수: 0 / 0.2 / 0.5 / 1.0 kg), `payload.offset = 0.05 m`
  - 총질량 = 드론(0.375) + 화물, 관성은 평행축 정리로 보정
- **호버 구간 확보 (`controlParams.m`):** `fail_time`을 0 → **5.0초**로 변경.
  - 0~5초: 로터 4개 정상 호버 (무게 측정 구간)
  - 5초~: 로터 1,3번 고장 → 스핀 안정화 비행

### 핵심 물리 검증 (연구 동기 입증)
- 화물 **0g**: 고장 후에도 고도 유지 (정상 비행)
- 화물 **200g**: 고장 후 고도 하락 → **"화물 모르는 기존 FTC는 고장 시 화물 때문에 추락"** 재현 성공.
- → 적응형 FTC의 필요성을 시뮬레이션으로 입증. (최종 보고서 핵심 그래프 후보)

### 무게 추정 알고리즘 (`estimate_payload.m`)
- **원리:** 수평 호버에서 IMU 비력 `accB_z = -T/m` → **`m = T / |accB_z|`**
  - 추력 T는 측정 로터속도로 독립 계산: `T = Σ k0·wᵢ²`
  - 화물무게 = 추정 총질량 − 드론 자체무게(0.375)
- **방식:** 후처리(post-processing). PX4 Hover Thrust Estimator 로직 참조.

### 1차 결과 (화물 참값 200g)
| 추정 구간 | 추정 화물무게 | 오차 | 표준편차 |
|------|------|------|------|
| 2.0~4.5초 | 233 g | +33 g (16.7%) | 0.0161 kg |
| **2.0~2.5초 (안정 구간만)** | **207 g** | **+7 g (3.3%)** | **0.0040 kg** |

→ 구간을 안정 호버 초반으로 좁히자 **오차 16.7% → 3.3%, 표준편차 1/4로 감소.** 무게추정 1차 프로토타입 검증 완료.

### 화물 조건별 검증 (추정 구간 2.0~2.5초)

| 화물 참값 | 추정 총질량 | 추정 화물무게 | 오차 | 오차율 |
|------|------|------|------|------|
| 0 g | 0.3797 kg | 5 g | +5 g | — (참값 0) |
| 200 g | 0.5815 kg | 207 g | +7 g | 3.3% |
| 500 g | 0.8838 kg | 509 g | +9 g | 1.8% |

→ 0 / 200 / 500g 모든 조건에서 **오차 10g 이내**로 추정 성공. **무게 무관하게 동작하는 추정기 검증 완료.** (화물 없을 땐 ≈0g, 무거울수록 추력 신호가 커져 오차율 감소)

- **발견:** 호버 안정 구간(t≈2.0초)에서는 추정값이 참값(0.575kg)에 거의 정확. 시간이 지나며 드론이 미세하게 움직이면 간이 추력모델(`k0·w²`)과 실제 복잡 공력모델의 차이가 커져 오차 증가.
- **통찰:** **추정 정확도는 호버 안정성에 민감** → 안정 구간 선택이 핵심. (계획서의 "안정된 호버 구간에서 1회 추정" 방침의 근거)

## 실험 설계 (가설 · 검증 방법 · 검증 결과)

### 가설

- **주가설(H):** 화물 무게를 추정하여 제어기의 파라미터(질량·관성)와 게인·허용기울기를 화물 조건에 맞게 적응시키면, **고정 파라미터 기존 FTC가 추락하는 화물 조건에서도 안정 비행(생존)이 가능하다.**
- **세부 질문(분리 검증):**
  - **H1:** 질량(+관성)만 올바르게 반영해도 추락을 막을 수 있는가? (INDI 추력배분 G행렬 보정만으로 충분한가)
  - **H2:** 질량만으론 부족하고, 스핀 중 무게중심-원심력에 의한 기울기 심화 때문에 **게인·허용기울기(`par.chi`, `par.position_maxAngle`)까지** 적응해야 하는가?
- → H1/H2 중 무엇이 맞는지 가리는 것 자체가 핵심 연구 결과. (현재 미확정)

### 검증 방법

- **비교 대상 (동일 화물·동일 시나리오, 제어기만 교체):**
  - **B:** 기존 FTC — 질량 0.375 고정, 게인 고정 (화물 모름)
  - **C1:** 적응형(부분) — 추정 질량·관성만 반영
  - **C2:** 적응형(전체) — 질량·관성 + 게인 + 허용기울기 모두 적응
  - (계획서 4단계의 고정 PID는 하한 기준선으로 추가)
- **공통 시나리오:** 0~5초 정상 호버(이 구간에서 무게 추정) → 5초 로터 1,3번 고장 → 스핀 안정화
- **화물 조건:** 0 / 200 / 500 g (1000g은 추력 포화 가능성 별도 확인)
- **평가 지표:**
  1. **생존 여부** (추락 / 안정) — 1차 판정
  2. 고장 후 **최대 기울기** (롤·피치)
  3. **고도 손실 / 오버슛**
  4. 자세 **정착 시간**
  5. **최대 스핀 각속도**
- **전제(완료됨):** 무게 추정 정확도 — 200g→207g(3.3%), 500g→509g(1.8%)로 검증 완료. 적응에 쓸 추정값은 신뢰 가능.

### 검증 결과

> 3·4단계 진행하며 채움 (현재 B만 확인: 화물 200g에서 고장 후 추락).

| 화물 | 제어기 B (기존) | 제어기 C1 (질량만) | 제어기 C2 (전체 적응) |
|------|------|------|------|
| 0 g | 안정 (확인) | - | - |

### 수정 파일

**`simParams.m`** — 41번 줄 뒤에 화물 주입 블록 추가:
```matlab
%% Payload (적응형 FTC 연구용) ----------------------------------
payload.mass   = 0.5;     % [kg] 진짜 화물무게 (실험: 0 / 0.2 / 0.5 / 1.0)
payload.offset = 0.05;    % [m]  하단 고정 오프셋 (무게중심 아래)
simVars.payload = payload;
simVars.drone.massDry = loadedParams.parameters.m;          % 드론 자체무게(추정 기준)
simVars.drone.mass = simVars.drone.massDry + payload.mass;  % 총질량 = 드론 + 화물
simVars.drone.Iv   = simVars.drone.Iv + payload.mass*payload.offset^2*diag([1 1 0]); % 평행축
% --------------------------------------------------------------
```
→ plant(EOMQuat)가 읽는 `simVars.drone.mass`/`.Iv`에 화물 반영. 제어기는 모름.

**`controlParams.m`** — 15번 줄 고장 시점 변경:
```matlab
par.fail_time = 0.0;  →  par.fail_time = 5.0;   % 0~5초 호버 구간 확보(무게추정용)
```

### 추정기 핵심 로직 (`estimate_payload.m`)
- `tsmat()` 헬퍼: Simulink Timeseries 데이터를 `[시간×채널]`로 자동 정렬
- `T = Σ k0·wᵢ²` (측정 로터속도 `wRotorMeas`로 추력 독립 계산)
- `interp1`로 추력·가속도 시간축 정렬 → `m = T/|accB_z|` → 호버 구간 평균
- 참값 대비 오차/오차율 자동 출력 + figure 200 시각화
